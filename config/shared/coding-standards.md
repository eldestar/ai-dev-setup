# Shared Coding Standards

- Keep scripts readable and boring.
- Prefer explicit detection over assumptions about paths, shells, or package managers.
- Make every write idempotent.
- Back up existing files or ask before replacement.
- Log actions, skipped items, and manual fallbacks.
- Validate external tools with their own version commands.
- Keep install commands close to official vendor recommendations.
- Avoid user-specific absolute paths.
